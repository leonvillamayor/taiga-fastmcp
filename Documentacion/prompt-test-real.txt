Tenemos un proyecto que implementa un servidor/cliente mcp, que utiliza uv para gestionar el proyecto y los entornos virtuales,  que permite interactuar con taiga. Necesito
que  verifiques el codigo del proyecto, fichero a fichero. El objetivo es que el proyecto lea las credenciales
incluidas en el fichero .env, se conecte y que cada herramienta haga lo que se pide.

Queremos refactorizar el codigo para que se utilicen buenas practicas de programacion python, DDD (Domain Driven Desing), previo uso de TDD (Test Driven Development),
donde los test estaran en rojo y el codigo debera implementar lo necesario para que los test aparezcan en verde.
Como queremos implantar herramientas con fastmcp que utilicen uno a uno todos los servicios que ofrece taiga mediante su api, analizaremos el fichero Documentacion/taiga.md de forma
exhaustiva y minuciosa para extraer todas las funcionalidades que taiga ofrece mediante su api y que deben de ser usables mediante herramientas mediante nuestra implantacion con fastmcp.

Los test deben de ser exhaustivos para garantizar que la logica de negocio cumple con los requerimientos.

Como utilizamos el framework fastmcp, debemos de utilizar todas las facilidades que ofrece este framework para que
la definición del servidor y cliente mcp siga las buenas practicas.
